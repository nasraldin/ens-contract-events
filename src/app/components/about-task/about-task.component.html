ENS Contract Events by: Nasr Eldin

Task Overview

We Need to investigate or list the ENS contract events of the past days. So, we will write a MVP (minimum viable product) DApp to get a list of the ENS contract events of the past (e.g. the last 1 or 2 days) focusing on the event for registered names.

NameRegistered (string name, bytes32 label, address owner, uint256 cost, uint256 expires)

Terminology

ENS

ENS is the Ethereum Name Service, a distributed, open, and extensible naming system based on the Ethereum blockchain.

The ENS offers a secure and decentralised way to address resources both on and off the blockchain using simple, human-readable names.

DApp

A decentralized application (DApp, dApp, Dapp, or dapp) is a computer application that runs on a distributed computing system. DApps have been mostly popularized by distributed ledger technologies (DLT), namely the Ethereum Blockchain, where DApps are often referred to as smart contracts.

DApps may run on top of distributed computing systems such as Ethereum or Bitcoin. Decentralized applications are stored on and executed by a Blockchain system.

Ethereum address

Is simply a 64-character hex string generated subject to various rules defined in the Ethereum yellow paper.

Smart Contract

A smart contract is a computer protocol intended to digitally facilitate, verify, or enforce the negotiation or performance of a contract.

Scope of Work

Display contract events for past 1,2 days in the table UI and can search, filter, sort events with pagination.

For more info about project stories and status you can visit link porject dashboard

https://github.com/nasraldin/ens-contract-events/projects/1

Initially, we planned to implement this set of features to be simple yet useful:

  • Store ABI, address, provider records in the json and load contract info within app initialize.
  • Quickly switch between networks: Mainnet/Ropsten/etc or your own localhost testnet.
  • Client app interact with smart contract and get past events from contract.
  • User-friendly search within contract events
  • Simple & short steps to view events
  • Excellent performance
